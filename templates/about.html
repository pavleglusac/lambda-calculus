What is lambda calculus?

Lambda calculus is a formal mathematical system for describing function. Lambda calculus can only describe functions and their applications. That's it. Using such basic concepts, it's possible to write any program, just like with a regular programming language. It is also called 'the assembly of functional languages'. Lambda calculus is equivalent to the Turing machine, which is proven by the Church-Turing thesis.


Functional programming languages like Haskell are based on LC. In fact, Haskell is based on System F (second-order lambda calculus). Functional programming languages are immutable, efficient, and more 'bug-free' than the regular imperative programming languages.

Notation

The symbol λ is used to denote the start of a function definition. For example:

λx.x

This is a function definition. It takes one parameter, x. The dot represents beginning of the function body. The aforementioned function takes in one parameter, x, and evaluates to that parameter.

Grammar and evaluation

The grammer that describes lambda calculus is fairly simple.

Expression ::= Variable | Application | Abstraction
Variable ::= ID
Application ::= (Expression Expression)
Abstraction ::= λID.Expression

β-reduction

β-reduction is a computational step in our language. It's a rule that tells us how to evaluate our functions. It subtitutes the function parameter with the appropriate expression. Let's say, to reduce (λn.n × 2) 7, we have:

(λn.n × 2) 7

(λ7.7 × 2)

(7 × 2)

Free and bound variables

Each appearance of a variable is either free to λ or bound to λ. β-reduction can replace the bound variables once parameters are applied, but not the free variables.

α-conversion

α-conversion is a process of changing bound variables' names. For example, α-conversion of λx.x might yield λy.y. This is not a trivial process, because the varaible names must be carefully chosen so that they don't shadow other names. The best way to solve this problem is using the De Brujin index for name resolution. It guarantees that any two α-equivalent terms are syntactically identical.

Numbers

Number representation via functions looks like this:

0 := λf.λx.x

1 := λf.λx.f x

2 := λf.λx.f (f x)

3 := λf.λx.f (f (f x))

and so on. Successor function can be used to get the same effect:

SUCC := λn.λf.λx.f (n f x)

Using that, we can easily define addition. We just need to apply succession the required number of times.

PLUS := λm.λn.λf.λx.m f (n f x)

Thus, we can define multiplication like adding the same parameter m times to zero:

MULT := λm.λn.m (PLUS n) 0

and so on. Using the previous abstractions we can build more algebraic functions.

Logic

To define any logical operator we first must define boolean values True and False.

TRUE := λx.λy.x

FALSE := λx.λy.y

Using that, we can define the standard logic operators like:

AND := λp.λq.p q p

OR := λp.λq.p p q

NOT := λp.p FALSE TRUE
